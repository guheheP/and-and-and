## 次期フェーズ

### Phase 3: 自動アップデート + 季節イベント（v0.5.1）

#### 3-1. 自動アップデート機能
- **優先度:** 高
- **概要:** GitHub Releasesを配布元とし、起動時にバックグラウンドで更新チェック。更新があればトースト通知し、ユーザーの選択で再起動して適用。
- **技術スタック:**
  - `electron-updater` パッケージ（electron-builderに同梱）
  - GitHub Releasesにビルド成果物をアップロード（`.exe` / `.app`）
  - `autoUpdater` APIでチェック・ダウンロード・インストール
- **実装内容:**
  - **main.js（Electronメインプロセス）:**
    - `autoUpdater.checkForUpdates()` を起動時に呼び出し
    - `update-available` / `update-downloaded` イベントをIPC経由でレンダラーに通知
    - `autoUpdater.quitAndInstall()` をIPC経由で呼び出し可能に
  - **preload.js:**
    - `onUpdateAvailable(callback)` / `onUpdateDownloaded(callback)` を追加
    - `installUpdate()` を追加
  - **レンダラー側UI:**
    - ステージ下部 or タイトルバーにトースト通知「新バージョンあり — 更新して再起動」
    - バージョン情報モーダルに現在のバージョン表示 + 「更新を確認」ボタン
  - **package.json:**
    - `electron-updater` を依存に追加
    - `build.publish` に GitHub provider 設定
  - **開発時の除外:** `app.isPackaged` チェックにより開発環境では更新チェックをスキップ
- **影響ファイル:** main.js, preload.js, src/js/main.js, src/index.html, package.json
- **見積もり:** 小〜中規模

#### 3-2. 季節イベントシステム
- **優先度:** 中
- **概要:** 実際の日付に連動した季節限定イベント
- **実装内容:**
  - 春（3-5月）: 桜吹雪、花粉（デバフ+ボーナス）
  - 夏（6-8月）: 花火大会、猛暑（成長加速）、かき氷
  - 秋（9-11月）: 紅葉、ハロウィン、収穫祭（全作物ポイント×2）
  - 冬（12-2月）: クリスマス強化、正月（おみくじ）、バレンタイン
  - 季節限定作物（各季節1〜2種）
  - 季節限定コスメ（季節イベントクリアで解放）
- **影響ファイル:** event-data.js, event-system.js, renderer-3d-events.js, master-data.js, achievement-system.js

## 技術的留意点

### 既存アーキテクチャとの整合性
- ES Modules + Vanilla JS パターンを維持（バンドラー不使用）
- Three.js CDN読み込みを維持
- localStorage セーブを維持（データ量増加に注意）
- 210×220px ウィンドウサイズの制約内でUI設計

### 後方互換性
- `loadState()` のマイグレーション処理で新フィールドをデフォルト値で補完
- `fieldState` → `fieldSlots` 移行済み
- 旧人型キャラID → `human` 自動変換済み

### 自動アップデートの前提
- GitHub リポジトリがパブリック（またはトークン設定済み）
- electron-builder でコード署名なしでも動作（Windows: NSIS unsigned可、macOS: 署名推奨）
- 開発中は `app.isPackaged === false` で更新チェックをスキップ
