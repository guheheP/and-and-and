# 実装計画 v0.5 — ボリューム拡張ロードマップ

> **作成日:** 2026-04-13  
> **現行バージョン:** 0.4.3  
> **方針:** ゲーム性を維持しつつ、プレイヤーが触れるコンテンツ量を大幅に増やす

---

## 現状分析

### 強み（維持すべき点）
- 放置ゲームとしてのコアループが完成している
- プレステージによるメタ進行が機能している
- 3Dボクセル描画が独自性を持っている
- 210×220pxの小窓に収まるミニマルUI

### ボリューム不足の領域
- **作物:** 通常10種 + イベント2種 → 種類が少なく、中盤以降に新鮮味がない
- **キャラクター:** 6体のみ、異形キャラ（仕様書記載）が未実装
- **イベント:** 14種 → パターン認識されやすい
- **エンドコンテンツ:** Lv.100以降の目標がない
- **音:** BGM/SEが一切ない → 常駐アプリとしての存在感が薄い
- **畑:** 1スロットのみ → 戦略性がない
- **ソーシャル/比較:** 完全ソロ、達成を共有する手段がない

## フェーズ別実装計画

### Phase 1: サウンド基盤 + 作物拡張（v0.5.0）
> 最も体感に影響する要素を先に

#### 1-1. BGM・SE システム
- **優先度:** 高
- **概要:** Web Audio API ベースのサウンドマネージャー
- **実装内容:**
  - `src/js/sound-manager.js` 新規作成
  - BGM: 環境音ループ（鳥のさえずり・風の音）をプロシージャル生成
    - OscillatorNode + GainNode で簡易シンセ（アセット不要）
    - 天候イベント時にBGM変化（雨音、雷鳴等）
  - SE: 収穫音、ガチャ演出音、レベルアップ音、実績解除音
    - 短いビープ/チップチューン系をプロシージャル生成
  - 音量設定（BGM/SE個別）+ ミュート切り替え
  - ツールバーにスピーカーアイコン追加
- **影響ファイル:** sound-manager.js(新規), main.js, ui-controller.js, index.html, main.css
- **見積もり:** 中規模

### Phase 2: 異形キャラクター + 畑拡張（v0.5.1）
> プレイヤーの選択肢と戦略性を増やす

#### 2-1. 異形キャラクター実装（6体）
- **優先度:** 中
- **概要:** 仕様書にあった異形キャラをCHARACTER_MASTERに正式登録
- **追加キャラ:**

| ID | 名前 | 説明 | 解放条件 |
|----|------|------|---------|
| dog | 犬 | 柴犬風 | 実績「犬の訪問を5回目撃」 |
| cat | 猫 | 三毛猫風 | 実績「猫の訪問を5回目撃」 |
| robot | ロボット | メカ農夫 | プレステージ10回 |
| alien | エイリアン | 緑肌 | 累計100万ポイント |
| pumpkinhead | カボチャ頭 | ハロウィン | カボチャLv.50 |
| snowman | 雪だるま | 冬の農夫 | 雪イベントを10回目撃 |

- **3Dモデル:** dog/cat は四足歩行モデル、他は人型のバリエーション
- **新規実績:** 上記解放条件に対応する実績6種を追加
- **影響ファイル:** master-data.js, renderer-3d-models.js, achievement-system.js, ui-modals.js

#### 2-2. 複数畑スロット（最大3スロット）
- **優先度:** 中
- **概要:** 畑を最大3面まで拡張、同時に複数作物を栽培
- **実装方針:**
  - `fieldState` を配列化: `fieldSlots: [fieldState, fieldState, fieldState]`
  - プレステージアップグレード `fieldSlot2`, `fieldSlot3` で解放
  - 3Dシーンでは畑を横に並べて表示
  - ゲームループは各スロットを独立にtick処理
  - カメラ位置の微調整（3畑表示に対応）
- **影響ファイル:** game-state.js, game-loop.js, renderer-3d.js, renderer-3d-crops.js, prestige-data.js, ui-modals.js
- **見積もり:** 大規模（コア構造の変更）

### Phase 3: 季節イベント + 図鑑強化（v0.5.2）
> コンテンツの奥行きと収集要素

#### 3-1. 季節イベントシステム
- **優先度:** 中
- **概要:** 実際の日付に連動した季節限定イベント
- **実装内容:**
  - 春（3-5月）: 桜吹雪、花粉（デバフ+ボーナス）
  - 夏（6-8月）: 花火大会、猛暑（成長加速）、かき氷
  - 秋（9-11月）: 紅葉、ハロウィン、収穫祭（全作物ポイント×2）
  - 冬（12-2月）: クリスマス強化、正月（おみくじ）、バレンタイン
  - 季節限定作物（各季節1〜2種）
  - 季節限定コスメ（季節イベントクリアで解放）
- **影響ファイル:** event-data.js, event-system.js, renderer-3d-events.js, master-data.js, achievement-system.js

#### 3-2. 図鑑・コレクション強化
- **優先度:** 中
- **概要:** 現在の簡素なカタログを図鑑システムに昇格
- **実装内容:**
  - 作物図鑑: 3Dモデルプレビュー、累計収穫数、最高レベル、発見日
  - キャラクター図鑑: 3Dモデル回転プレビュー、解放条件表示
  - イベント図鑑: 発生回数ランキング、レアイベント遭遇率
  - コンプリート率（%）表示 → 実績と連動
- **影響ファイル:** ui-modals.js, ui-controller.js, game-state.js, index.html, ui.css

### Phase 4: エンドコンテンツ + 統計（v0.6.0）
> 長期プレイヤーへの目標提供

#### 4-1. チャレンジモード
- **優先度:** 低〜中
- **概要:** 特殊条件下でのクリア目標

  - チャレンジクリアで限定コスメ獲得
  - チャレンジ専用のミニGameState（本編に影響しない）
- **影響ファイル:** challenge-system.js(新規), game-state.js, ui-modals.js

#### 4-2. 統計ダッシュボード
- **優先度:** 低
- **概要:** プレイ統計の可視化
- **実装内容:**
  - 総プレイ時間、総収穫回数、作物別収穫数グラフ
  - 1日あたりのポイント推移
  - プレステージ履歴（各回のレベル・通貨）
  - Canvas 2Dでシンプルなグラフ描画
- **影響ファイル:** stats-dashboard.js(新規), game-state.js, ui-modals.js

#### 4-3. プレステージ2段階目（超越）
- **優先度:** 低
- **概要:** プレステージのさらに上位のメタ進行
- **条件:** プレステージ20回以上 + レベル100到達
- **リセット対象:** プレステージ通貨・アップグレードも含む全リセット
- **獲得:** 超越通貨 → 超越アップグレード（基礎倍率の恒久強化）
- **影響ファイル:** prestige-data.js, game-state.js, ui-modals.js, prestige.css

## 実装優先順位まとめ

| 順位 | 項目 | フェーズ | 理由 |
|------|------|---------|------|
| 1 | BGM・SE | Phase 1 | 体験の質に直結。アセット不要で実装可能 |
| 2 | 作物15種追加 | Phase 1 | 最もコスト対効果が高い。中盤以降の目標提供 |
| 3 | 異形キャラ6体 | Phase 2 | 仕様書の残件消化 + コレクション要素 |
| 4 | 複数畑スロット | Phase 2 | 戦略性の追加。ただしコア構造変更のためリスクあり |
| 5 | 季節イベント | Phase 3 | 長期継続のモチベーション |
| 6 | 図鑑強化 | Phase 3 | 既存データの活用、UI改善 |
| 7 | チャレンジモード | Phase 4 | エンドコンテンツ、開発コスト高め |
| 8 | 統計ダッシュボード | Phase 4 | あると嬉しいが必須ではない |
| 9 | 超越システム | Phase 4 | 最終盤コンテンツ、Phase 1-3の実装後に検討 |

## 技術的留意点

### 既存アーキテクチャとの整合性
- ES Modules + Vanilla JS パターンを維持（バンドラー不使用）
- Three.js CDN読み込みを維持
- localStorage セーブを維持（データ量増加に注意）
- 210×220px ウィンドウサイズの制約内でUI設計

### 後方互換性
- `loadState()` のマイグレーション処理で新フィールドをデフォルト値で補完
- `fieldState` → `fieldSlots` への移行はマイグレーション関数で対応
- 新作物・新キャラは既存セーブデータに影響しない（マスターデータ追加のみ）

### パフォーマンス
- 3畑表示時のThree.jsオブジェクト数増加に注意
- 作物25種の3Dモデルは初回生成→キャッシュする設計
- サウンドはプロシージャル生成でファイルサイズゼロ
