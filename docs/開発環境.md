# 開発・環境構築ガイドライン

本ドキュメントは、デスクトップ常駐型アプリ（Electronベース）の開発環境、技術スタック、およびデプロイメントに関する規定をまとめたものです。

## 1. 開発フェーズと環境の分離

テストプレイとUI開発の容易さを確保するため、フロントエンド単体での動作確認環境と、Electronによるデスクトップアプリ化環境を分けて運用します。

### フェーズ1: ブラウザ環境（フロントエンド開発）
* **目的:** UIのレイアウト、フロントエンド側のアニメーション・基本挙動の検証。
* **確認環境:** GitHub Pages または ローカルの開発用Webサーバー
* **制限事項:** ブラウザ上で動作させるため、OSネイティブなAPI（ウィンドウの背景透過、最前面固定、OS全体のグローバル入力検知など）は機能しません。

### フェーズ2: Electron環境（デスクトップアプリ統合）
* **目的:** 常駐アプリとしての必須要件（透過、最前面表示、バックグラウンドでのキー・マウス入力検知）の実装と最終ビルド。
* **対象OS:** Windows / macOS
* **配布形式:** 実行ファイル（`.exe` / `.app` 等）としてパッケージング。

## 2. 技術スタック・必須ツール

* **実行環境:** Node.js
* **アプリケーションフレームワーク:** Electron
* **フロントエンド:** HTML, CSS, JavaScript
* **バージョン管理:** Git, GitHub
* **パッケージマネージャー:** npm または yarn
* **ビルド・パッケージング:** `electron-builder`

## 3. ディレクトリ構成（推奨）

フロントエンド用のファイル群と、Electron本体のプロセス（Mainプロセス）を明確に分離して管理します。

* `src/` : フロントエンド関連（GitHub Pages等で公開・テストする領域）
  * `index.html` : UIのメインファイル
  * `css/` : スタイルシート
  * `js/` : フロントエンドのロジック
  * `assets/` : 画像・音声などの静的リソース
* `main.js` : ElectronのMainプロセス（ウィンドウ生成、OSネイティブ機能の制御）
* `package.json` : 依存ライブラリ、各種スクリプト（起動・ビルド）の定義
* `.gitignore` : `node_modules/` やビルド成果物など、Git管理外とするファイルの指定

## 4. ビルドと配布に関する規定

* **Webブラウザでの確認:** `src/` ディレクトリ配下のファイルをブラウザで読み込む。
* **ローカルでのアプリ起動:** `npm start` (Electronによる `main.js` の起動)。
* **アプリのビルド:** `electron-builder` を使用し、テスト・配布用の実行ファイルを生成する。
