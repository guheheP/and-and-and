# 総合企画・仕様書：作業に寄生する常駐型インフレゲーム（農業テーマ）

---

## 【第1部】ゲーム企画書（コンセプト・ゲームデザイン）

### ■ コンセプト
* **作業を邪魔しないのに、気づいたら依存している常駐型インフレゲーム。**
* プレイヤーが操作せずとも、キャラクターが自動で種植えから収穫までを行い、ポイントが増加していく。
* デスクワーク等の集中作業の合間に、視界の端でキャラクターや状態が勝手に成長していく感覚と、たまに触ることで一気に加速するカタルシスを提供する。

### ■ ゲーム概要
* キャラクターが自動で農作業（種植え〜収穫）を行う。
* 収穫時にポイントが獲得される。
* プレイヤーは主にガチャやキャラクターの変更で間接的に影響を与える。

### ■ 作物システム
* ガチャにより作物の種を入手する。
* 入手した種はランダム、またはプレイヤーの選択で植えられる。
* 種を所持していない場合は、デフォルトで作物（例：トマト）が植えられる。

**作物パラメータの例：**
各作物には以下のような個性を持たせる。
* `収穫までの時間` / `獲得ポイント量` / `成長速度`
* （例）収穫が遅いが高ポイント / 収穫が早いが低ポイント / レア度による獲得ポイントの増減

### ■ 成長・進行要素
* 作物を育て続けることでレベルが上昇する。
* レベル上昇によりポイント生成量が増加する。
* 作物はレベル到達でガチャ専用の作物も常設解放される。

### ■ キャラクター・カスタマイズ要素
作物を育てるキャラクターの存在により、視覚的な楽しさと愛着を強化する。

* **人型キャラクター:** 成人男性、成人女性、少女、少年、おじいさん、おばあさん
* **異形キャラクター:** 犬、猫、木、鳥、昆虫（クワガタなど）
    * 現実ではありえない存在が農作業を行うことで、ユーモアと独自性を演出する。
* **着せ替え:** キャラクターのカラーリング等を変更可能にし、プレイヤーの愛着や継続率を向上させる。

## 【第2部】システム仕様書（アーキテクチャ・データ構造・実装要件）

AIアシスタント向けの実装ガイドラインおよびデータ定義。

### 1. アーキテクチャ概要
* **実行環境:** Electron (Windows / macOS)
* **フロントエンド:** HTML, CSS (SCSS推奨), Vanilla JavaScript
* **描画方式:** 画像アセット非使用。DOM要素とCSSの組み合わせ（長方形ベース）によるピクセルアート風レンダリング。
* **ウィンドウ制御:** 背景透過 (`transparent: true`)、フレームレス (`frame: false`)、最前面固定 (`alwaysOnTop: true`)。
* **マウス制御:** デフォルトはクリックスルー (`setIgnoreMouseEvents(true, { forward: true })`)。メニューやガチャなどの特定のUI要素ホバー時のみクリックを受け付ける仕様。

### 2. データモデル（State定義）

#### 2.1. GameState (セーブデータとして永続化する主状態)
```json
{
  "points": 0,               // 現在の所持ポイント
  "totalEarnedPoints": 0,    // 累計獲得ポイント（レベル計算用）
  "level": 1,                // プレイヤー/農場のレベル
  "seedsInventory": {        // 所持している種の数
    "tomato": 0,
    "potato": 2,
    "golden_apple": 0
  },
  "currentCharId": "uncle",  // 現在表示しているキャラクター
  "fieldState": {            // 畑の状態（単一または複数スロット）
    "isPlanted": false,
    "cropId": null,
    "plantedAt": null,       // 植えた時間 (UNIX Timestamp)
    "progress": 0            // 成長度 (0.0 ~ 1.0)
  }
}
```

#### 2.2. MasterData (読み取り専用のマスターデータ)
**CropMaster (作物データ)**
```json
{
  "tomato": { 
    "name": "トマト", 
    "growTimeMs": 5000,     // 収穫までの時間
    "basePoint": 10,        // 獲得ポイント量
    "rarity": 1, 
    "isDefault": true       // 種がない場合のデフォルト
  },
  "potato": { 
    "name": "ジャガイモ", 
    "growTimeMs": 15000, 
    "basePoint": 50, 
    "rarity": 2, 
    "isDefault": false 
  }
}
```

**CharacterMaster (キャラクターデータ)**
* **ID定義:** `man`, `woman`, `girl`, `boy`, `grandpa`, `grandma`, `dog`, `cat`, `tree`, `bird`, `insect`
* **実装要件:** レンダリングは全てCSSクラスの切り替え (`.farmer--dog` など) とCSS変数 (`--color-body` など) の注入で行う。

### 3. ゲームループ（Tick処理）

JavaScriptの `requestAnimationFrame` または `setInterval` を用いて、毎秒以下の自動農作業フローを実行する。

1.  **種植えフェーズ:**
    * `fieldState.isPlanted === false` の場合実行。
    * `seedsInventory` を確認し、種を消費して植える。種がない場合は強制的にデフォルト作物（`tomato`）をセット。
    * `plantedAt` に現在時刻をセット。
2.  **成長フェーズ:**
    * `fieldState.isPlanted === true` の場合実行。
    * 現在時刻と `plantedAt` の差分から `progress` (0.0 ~ 1.0) を計算。
    * `progress` に応じてUI（作物のCSSサイズ等）を更新。
3.  **収穫フェーズ:**
    * `progress >= 1.0` に達した場合実行。
    * `CropMaster` の `basePoint` にレベル倍率等を掛けた値を `points` に加算。
    * `fieldState` をリセットし、再び1(種植え)へループする。

### 4. 成長・進行システム
* **レベルアップ条件:** `totalEarnedPoints` が閾値を超える。
* **恩恵:** 収穫時ポイントの倍率ボーナス付与、およびガチャプールへの新規作物IDの常設解放。

### 5. アニメーション仕様
* **待機時:** CSS `keyframes` による緩やかな拡縮（呼吸アニメーション）。
* **作業中（種植え・収穫）:** JavaScriptから `.is-working` クラスを一時的に付与し、CSSでクワを振る（前後に動く）アニメーションを発火させる。
* **要件:** 画像アセット非依存のため、全ての状態変化はDOMのクラス・スタイル操作のみで完結させること。
