## 【第1部】ゲーム企画書（コンセプト・ゲームデザイン）

### コンセプト

* **作業を邪魔しないのに、気づいたら依存している常駐型インフレゲーム。**
* プレイヤーが操作せずとも、キャラクターが自動で種植えから収穫までを行い、ポイントが増加していく。
* デスクワーク等の集中作業の合間に、視界の端でキャラクターや状態が勝手に成長していく感覚と、たまに触ることで一気に加速するカタルシスを提供する。

### ゲーム概要

* キャラクターが自動で農作業（種植え〜収穫）を行う。
* 収穫時にポイントと経験値が獲得される。
* プレイヤーはガチャ・キャラクターカスタマイズ・プレステージで間接的にゲーム進行に影響を与える。
* ランダムイベント（天候・来客・贈り物）により変化が生まれる。

### コアループ

```
種植え → 成長 → 収穫 → ポイント/EXP獲得
    ↓                         ↓
  ガチャで種入手          レベルアップ（新作物解放）
                              ↓
                    プレステージ（永続強化）
```

### 4. ゲームループ（Tick処理）

`requestAnimationFrame` による毎フレーム処理。

#### 4.1. 種植えフェーズ

1. `fieldState.isPlanted === false` の場合実行
2. 優先作物（`selectedCropId`）があれば、その種を消費して植える
3. なければ `seedsInventory` からランダムに選択
4. 種がない場合はデフォルト作物（トマト）を植える
5. 無限化済み作物（トマト or 作物Lv.100以上）は種を消費しない

#### 4.2. 成長フェーズ

1. 現在時刻と `plantedAt` の差分から `progress` を計算
2. 成長時間の短縮要素:
   - プレステージ `growthSpeed` 効果（最大90%短縮）
   - イベント `growthBoost` 倍率（雨: ×3, 大雨: ×5, ダイヤの雨: ×10 等）
3. 3Dモデルの蔓・葉・果実の表示を `progress` に応じて更新

#### 4.3. 収穫フェーズ

`progress >= 1.0` で発動。ポイント計算式:

```
獲得ポイント = basePoint
  × getPointMultiplier(level)        // レベル倍率: 1.0 + (lv-1)×0.03
  × getCropLevelMultiplier(cropLv)   // 作物レベル倍率: 1.0 + (cropLv-1)×0.01
  × prestigeBasePoints効果           // プレステージ: 1 + lv×0.5
  × eventPointBoost                  // イベント効果（鳥のフン等）
  × luckyMultiplier                  // ラッキー収穫: ×3 (確率: prestigeLv×3%)
```

```
獲得EXP = baseExp
  × getCropLevelMultiplier(cropLv)
  × prestigeExpMultiplier効果
  × eventBoost
  × luckyMultiplier
```

#### 4.4. 自動処理

| 処理 | 間隔 | 条件 |
|------|------|------|
| オートセーブ | 10秒 | 常時 |
| オートガチャ | 5秒 | プレステージ `autoGacha` 解放時 |
| 実績チェック | 10秒 | 常時 |
| イベント判定 | 60秒 | 常時 |

### 5. レベルシステム

#### 5.1. プレイヤーレベル

- **閾値計算:** `Math.floor(100 × (1.08^(level-1) - 1) / 0.08)`
- **ポイント倍率:** `1.0 + (level - 1) × 0.03`（Lv.100 = ×3.97）
- **作物解放:** レベル到達で新しい作物がガチャプールに追加

#### 5.2. 作物レベル

- **EXP蓄積:** 収穫ごとに作物EXPが1加算（`cropExpBoost` で増加）
- **レベル計算:** `Math.floor((1 + √(1 + 8×exp)) / 2)`
- **ポイント倍率:** `1.0 + (cropLv - 1) × 0.01`（Lv.100 = ×1.99）
- **無限化:** 作物レベルが100に達すると種を消費しなくなる（トマトは初期から無限）

### 7. プレステージシステム

#### 7.1. 転生条件・通貨

- **最低レベル:** 50
- **通貨計算:** `max(1, floor((level - 45)^exponent))`
  - exponent: `min(2.5, 1.5 + prestigeCount × 0.05)`
- **リセット対象:** ポイント, レベル, EXP, 種, 畑状態
- **保持対象:** キャラクター設定, プレステージ通貨・アップグレード, 実績, コスメ

### 10. キャラクター・カスタマイズ

#### 10.1. 3Dボクセルモデル

画像アセット非使用。Three.jsのBoxGeometry/CylinderGeometry/SphereGeometryの組み合わせによるボクセル風レンダリング。

**モデル構成パーツ:** head, hair, eyes(×2), body, armL, armR, legL, legR

#### 10.2. カスタマイズ要素

- **ベースキャラクター:** 6体から選択
- **帽子:** 11種（実績報酬で解放）
- **アクセサリ:** 9種（実績報酬で解放）
- **カラー変更:** 肌・服・髪・パンツ・目のRGB個別指定 or ランダム生成
- **プリセット:** 5スロットに外見設定を保存/呼び出し

#### 10.3. カラー生成アルゴリズム

`generateRandomColors()` — HSLベースのランダム生成:
- 肌色: Hue 0〜30°
- 服: 自由
- 髪: 3パターン（自然色 / 黒系 / カラフル）
- パンツ・目: 自由

### 11. 描画・アニメーション

#### 11.1. Three.jsシーン構成

- **カメラ:** 正投影カメラ（frustum: 4）
- **ライティング:** AmbientLight(0.95) + DirectionalLight(1.0)
- **地面:** 草(0x5a9e3c) + 土(0x6b4226)
- **畑:** 杭×4本 + 蔓(10セグメント) + 葉(8対) + 果実(4個)
- **キャラクター:** ボクセル人型モデル

#### 11.2. アニメーション

| 状態 | アニメーション |
|------|-------------|
| 待機 | 呼吸（緩やかな拡縮） |
| 作業中 | 腕振りモーション（150ms） |
| 収穫時 | お辞儀→ジャンプ→バンザイ（450ms） |
| 収穫エフェクト | +Xpt フロート表示（1秒） |
| レベルアップ | フラッシュ演出（1.2秒） |
| 実績解除 | トースト通知（4秒） |

#### 11.3. イベント視覚エフェクト

- **雨/大雨/ダイヤの雨:** パーティクル落下
- **雪:** フレーク落下
- **雷雨:** 周期的なフラッシュ
- **入道雲:** 雲オブジェクト表示
- **台風:** 風エフェクト
- **贈り物/お友達:** 絵文字スプライトの横断表示
