# /review — コードレビューワークフロー

## 手順

1. **変更差分の確認**
   - `git diff` で変更内容を確認する
   - 変更の意図と影響範囲を理解する

2. **開発ルールとの整合性チェック**
   - `docs/DEVELOPMENT_RULES.md` のルールに違反していないか確認
   - 命名規則・コーディング規約に従っているか

3. **品質チェック**
   - [ ] エラーハンドリングは適切か
   - [ ] エッジケースが考慮されているか
   - [ ] セキュリティ上の問題はないか（ハードコードされた秘密情報等）
   - [ ] パフォーマンス上の問題はないか（N+1クエリ等）
   - [ ] 型安全性は確保されているか

4. **テスト確認**
   - 新機能に対応するテストが追加されているか
   - 既存テストが通るか確認する

5. **AImemory 確認**
   - 変更に関連する既知の問題が AImemory に記録されていないか確認
   - レビューで発見した知見を AImemory に記録すべきか検討

6. **レビュー結果の報告**
   - 問題点をリスト形式で報告する
   - 改善提案がある場合は具体的なコード例を添える
